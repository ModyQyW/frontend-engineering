# 选型

- 稳定：不会经常有破坏性更新，向后兼容度高，更新/迁移成本低
- 易用：使用相对简单，更有可能统一技术栈，更方便不同团队间人员流动
- 生态：需要考虑支持国内外生态，如小程序端、支付和分享场景等

<br>
<br>

<v-clicks>

- Angular：对小程序支持不佳
- React：心智负担较重，易用性不佳
- Solid、Fre 等：生态不成熟
- Vue：各方面相对不错

</v-clicks>

关于 React 的心智负担问题：

1. 原本 class 组件也就是类组件是相对完整的，function 组件也就是函数组件是有相当多限制的。

2. 引入 hooks 之后，函数组件开始趋向完整，虽然仍然有一些缺失，但已经是社区里的大趋势了。

4. 所以说心智负担较重。
